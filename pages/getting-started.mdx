## Development Environment Setup

<Steps>
### Clone Repository

```bash
git clone https://github.com/anima-unity/platform.git
cd platform
```

### Install Dependencies

<Tabs items={['npm', 'yarn', 'pnpm']}>
  <Tab>
    ```bash
    npm install
    ```
  </Tab>
  <Tab>
    ```bash
    yarn install
    ```
  </Tab>
  <Tab>
    ```bash
    pnpm install
    ```
  </Tab>
</Tabs>

### Configure Environment

Create a `.env` file in the project root:

```bash
cp .env.example .env
```

Required variables:

```plaintext
DATABASE_URL=postgresql://user:password@localhost:5432/anima
API_KEY=your_api_key_here
JWT_SECRET=your_jwt_secret
REDIS_URL=redis://localhost:6379
```

### Start Development Server

<Tabs items={['Local', 'Docker']}>
  <Tab>
    ```bash
    npm run dev
    ```
    Access the application at `http://localhost:3000`
  </Tab>
  <Tab>
    ```bash
    docker-compose up --build
    ```
    Access the application at `http://localhost:8080`
  </Tab>
</Tabs>

</Steps>

## Development Workflow

### Code Standards

<Callout type="info">
We follow strict coding standards to maintain consistency across the codebase.
</Callout>

```bash
# Run linter
npm run lint

# Run tests
npm run test

# Build application
npm run build
```

### Branch Strategy

- `main` - Production releases
- `develop` - Development branch
- `feature/*` - New features
- `bugfix/*` - Bug fixes
- `hotfix/*` - Production hotfixes

### Commit Convention

```bash
type(scope): subject

# Types: feat, fix, docs, style, refactor, test, chore
# Example: feat(auth): implement OAuth2 authentication
```

## API Integration

<Callout type="warning">
Ensure you have valid API credentials before starting integration.
</Callout>

```typescript
import { AnimaUnityClient } from '@anima-unity/sdk';

const client = new AnimaUnityClient({
  apiKey: process.env.API_KEY,
  environment: 'development'
});

// Example API call
const pets = await client.pets.list({
  limit: 10,
  offset: 0
});
```

## Troubleshooting

Common issues and solutions:

<Callout emoji="⚠️">
  **Database Connection Issues**
  
  If you cannot connect to the database, verify:
  1. PostgreSQL is running
  2. Database credentials are correct
  3. Database exists and is accessible
</Callout>

<Callout emoji="🔧">
  **Build Failures**
  
  If the build fails:
  1. Clear node_modules and reinstall
  2. Verify Node.js version
  3. Check for conflicting dependencies
</Callout>
