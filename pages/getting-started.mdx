# üêæ Getting Started with Anima Unity Microservices

Welcome to **Anima Unity** - Indonesia's first comprehensive microservices-based digital ecosystem for animal welfare, rescue, and empowerment. This developer guide will help you set up your microservices development environment and start building with our distributed platform.

<Callout type="info">
**"Tidak ada hewan yang dibiarkan tertinggal"** - No animal left behind. Our mission is to provide digital identity, access, protection, and care for every pet and wild animal through scalable microservices architecture, community collaboration, and compassionate technology.
</Callout>

## üöÄ Development Environment Setup

<Steps>
### Prerequisites Installation

Ensure you have the following tools installed:

```bash
# Container orchestration
docker --version          # >= 24.0
docker-compose --version  # >= 2.20

# Kubernetes (optional for local development)
kubectl version --client  # >= 1.28
minikube version          # >= 1.31

# Service mesh (optional)
istioctl version          # >= 1.19

# Development tools
git --version             # >= 2.40
node --version            # >= 18.0
java --version            # >= 17
python --version          # >= 3.9
go version                # >= 1.21
cargo --version           # >= 1.70
```

### Clone Microservices Repository

```bash
git clone https://github.com/anima-unity/microservices-platform.git
cd anima-unity-microservices
```

### Infrastructure Setup

<Tabs items={['Docker Compose', 'Kubernetes', 'Local Services']}>
  <Tab>
    ```bash
    # Start all infrastructure services
    docker-compose -f docker-compose.infrastructure.yml up -d
    
    # Verify infrastructure health
    docker-compose ps
    
    # View logs
    docker-compose logs -f
    ```
    
    Services started:
    - **API Gateway** (Kong): `http://localhost:8080`
    - **Service Discovery** (Consul): `http://localhost:8500`
    - **Message Broker** (RabbitMQ): `http://localhost:15672`
    - **Configuration Server**: `http://localhost:8888`
    - **Monitoring** (Prometheus): `http://localhost:9090`
    - **Observability** (Grafana): `http://localhost:3000`
    - **Distributed Tracing** (Jaeger): `http://localhost:16686`
  </Tab>
  <Tab>
    ```bash
    # Start local Kubernetes cluster
    minikube start --memory=8192 --cpus=4
    
    # Install Istio service mesh (optional)
    istioctl install --set values.defaultRevision=default -y
    kubectl label namespace default istio-injection=enabled
    
    # Deploy infrastructure
    kubectl apply -f k8s/infrastructure/
    
    # Check deployment status
    kubectl get pods -n anima-unity-system
    ```
  </Tab>
  <Tab>
    ```bash
    # Start services individually for development
    
    # PostgreSQL
    docker run -d --name postgres \
      -e POSTGRES_DB=anima_unity \
      -e POSTGRES_USER=anima \
      -e POSTGRES_PASSWORD=unity \
      -p 5432:5432 postgres:15
    
    # MongoDB
    docker run -d --name mongodb \
      -p 27017:27017 mongo:7
    
    # Redis
    docker run -d --name redis \
      -p 6379:6379 redis:7-alpine
    
    # RabbitMQ
    docker run -d --name rabbitmq \
      -p 5672:5672 -p 15672:15672 \
      rabbitmq:3-management
    ```
  </Tab>
</Tabs>

### Service Configuration

Create environment configuration for each service:

```bash
# Copy environment templates
cp .env.example .env.local

# Generate service-specific configurations
./scripts/generate-service-configs.sh
```

### Start All Microservices

<Tabs items={['Docker Compose', 'Individual Services', 'Kubernetes Deployment']}>
  <Tab>
    ```bash
    # Start all microservices
    docker-compose up --build -d
    
    # Check service health
    curl http://localhost:8080/health
    
    # View service logs
    docker-compose logs -f animaid-service
    ```
  </Tab>
  <Tab>
    ```bash
    # Start services individually in separate terminals
    
    # Terminal 1: AnimaID Service (Java/Spring Boot)
    cd services/animaid-service
    ./mvnw spring-boot:run
    
    # Terminal 2: AnimaVoice Service (Node.js/NestJS)
    cd services/animavoice-service
    npm run start:dev
    
    # Terminal 3: AdoptHub Service (Python/FastAPI)
    cd services/adopthub-service
    uvicorn main:app --reload --port 8003
    
    # Terminal 4: PawHelp Service (Java/Spring Boot)
    cd services/pawhelp-service
    ./mvnw spring-boot:run
    
    # Terminal 5: AnimaClinic Service (Node.js/Express)
    cd services/animaclinic-service
    npm run dev
    
    # Terminal 6: AnimaTrack Service (Rust/Actix)
    cd services/animatrack-service
    cargo run
    
    # Terminal 7: Community Service (Go/Gin)
    cd services/community-service
    go run main.go
    ```
  </Tab>
  <Tab>
    ```bash
    # Deploy to Kubernetes
    kubectl apply -f k8s/services/
    
    # Check deployment status
    kubectl get pods -n anima-unity
    kubectl get services -n anima-unity
    
    # Port forward for local access
    kubectl port-forward service/api-gateway 8080:80 -n anima-unity
    ```
  </Tab>
</Tabs>

</Steps>
